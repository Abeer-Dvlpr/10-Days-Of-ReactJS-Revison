# React 10-Day Learning Journey

Welcome to the complete React learning repo. This project contains 10 standalone day folders, each with a Vite React lesson and a matching `Day-XX.md` study note.

Each lesson is designed to be easy to follow, with examples, hooks, and real React concepts. Every day file now includes a navigation link at the bottom so you can move forward through the lessons quickly.

---
## 🚀 What’s included

- 10 day-by-day React lessons
- `Day-XX.md` notes in each lesson folder
- clean GitHub-ready structure after removing local build clutter
- fast navigation between lessons

## 📚 Lessons

1. [Day 01 — The Fundamentals](<./Day 01 — (The Fundamentals)/Day-01.md>)
   - React basics: components, JSX, props, lists, and rendering.
2. [Day 02 — State, Events & Conditional Rendering](<./Day 02 — (State, Events Conditional Rendering)/Day-02.md>)
   - useState, event handling, forms, and conditional UI.
3. [Day 03 — Side Effects & useEffect](<./Day 03 — (Side Effects,  useEffect)/Day-03.md>)
   - useEffect patterns, lifecycle behavior, cleanup, and data fetching.
4. [Day 04 — Refs & DOM Control](<./Day 04 — (Refs , DOM Control)/Day-04.md>)
   - useRef, DOM access, focus control, and mutable refs.
5. [Day 05 — Reducers & Performance Hooks](<./Day 05 — (Reducers,  Performance Hooks)/Day-05.md>)
   - useReducer, useMemo, useCallback, and React.memo.
6. [Day 06 — Context API](<./Day 06 — (Context API)/Day-06.md>)
   - context, prop drilling, provider patterns, and shared state.
7. [Day 07 — React Router](<./Day 07 — (React Router)/Day-07.md>)
   - client-side routing, routes, links, nested routes, and params.
8. [Day 08 — External State Management](<./Day 08 — (External State Management)/Day-08.md>)
   - Zustand, Redux Toolkit, global state, and store configuration.
9. [Day 09 — Advanced Patterns](<./Day 09 — (Advanced Patterns)/Day-09.md>)
   - custom hooks, compound components, and React Query ideas.
10. [Day 10 — Testing in React](<./Day 10 -(Testing)/Day-10.md>)
   - Jest, React Testing Library, rendering, events, and assertions.

## 💡 How to use this repo

1. Open one day folder.
2. Read the matching `Day-XX.md` file.
3. Use the button at the bottom of each note to continue to the next day.
4. Run `npm install` and `npm run dev` inside any day folder to launch the demo.
